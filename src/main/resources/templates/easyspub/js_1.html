script의 위치

HTML파일을 다 읽어야 웹페이지가 표시되고 script tag의 파일을 로드(다운)해야 기능이 적용된다
웹페이지를 열면 HTML파일을 로드하는 중에 script tag를 만나면 화면 로드가 중지되고 js로드가 진행된다
즉 화면에 보여지지 않은 상태에서 js가 로드되기를 기다려야 한다

사용자에게 화면을 우선적으로 보여줘야 할때 script tag를 body 하단에 두어 화면이 보인 후 js를 로드하게 한다
단점은 js 즉 기능이 로드전이라 먼저 보여지는 화면이 의미없는 화면일 수 있다

반대로 script tag를 header에 넣는 경우는 js 파일이 커 로드가 늦을 경우 화면에 아무것도 안보이는 현상이 발생한다

script tag의 옵션을 이용한 로드 순서
script async (default true) : HTML, script 병렬로 로드하지만 js가 로드가 완료되면 HTML로드와 관계없이 실행
script defer : HTML, script 병렬로 로드하지만 HTML로드가 완료되어야지만 js가 실행

script 실행 시점

javascript를 실행하는 일반적인 방법 : HTML 즉 화면이 전부 로드된 상태에서 진행

HTML이 전부 로드된 상태는 2가지로 구분
1. DOMContentLoaded: HTML즉 화면만 로드된 상태
2. window : HTML외 리소스(이미지 등등.. 미디어)도 로드된 상태

함수를 구성하는 방법 1. 이벤트에 익명함수 사용 : 재사용 못하는 함수
2. 이벤트에 일반함수 사용 : 함수의 재사용성
3. 이벤트 리스너에 익명함수 사용 : 재사용 못하는 함수
3. 이벤트 리스너에 일반함수 사용 : 함수의 재사용성

정리

javascript 로드 순서를 결정할때 header에 위치하던 body하단에 위치하던
javascript에서 이벤트로 처리할 것이므로 가독성이 높은데 적용한다.
javascript에서 시작 이벤트는 화면, 리소스가 전부 로드된 window를 기준으로 하고
코드 가독성이 좋은 상태를 선택한다.
화면에 진입했을때 사용하므로 시작함수로 익명함수를 사용하고
익명함수 안에 사용함수를 넣는게 좋을것 같다.
